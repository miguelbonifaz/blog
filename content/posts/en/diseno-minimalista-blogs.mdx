When it comes to technical blogs, the design should disappear. The content is the protagonist.

## The visual noise rule

Every element on screen that is **not content** is noise. Excessive borders, unnecessary gradients, and animations that add no value all distract the reader.

### Principles I follow

- **Clear typography**: A sans-serif font for body text, monospace for code
- **Enough contrast**: Light text on a dark background, without being harsh
- **Negative space**: Generous margins so content can breathe
- **Obvious hierarchy**: Headings should guide reading without effort

## Code as an example

A well-presented code block is worth more than a thousand words:

```css
body {
  background: #0a0a0a;
  color: #e5e5e5;
  font-family: system-ui, sans-serif;
  line-height: 1.7;
}
```

## What I avoid

1. Sidebars with unnecessary widgets
2. Aggressive subscription pop-ups
3. More than two typefaces
4. Saturated colors competing with code

> "Good design is as little design as possible." — Dieter Rams

Next time you design a technical blog, ask yourself: _does this element help the reader, or does it just make me feel good as a designer?_
