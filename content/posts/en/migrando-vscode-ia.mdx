After years of using VS Code as my main editor, I decided to explore a workflow centered on AI tools. Here is what I learned.

## The problem with the traditional workflow

My previous workflow looked like this:

1. Open VS Code
2. Find the right file
3. Read the code and understand the context
4. Make the change
5. Test manually
6. Repeat

Step 3 was where I lost the most time. Understanding context in a large project can take longer than the change itself.

## The new workflow

With tools like Claude Code, the flow changes:

```bash
# Describe the intent
claude "Add error handling to the payment webhook"

# Claude reads, understands, and proposes changes
# I review and approve
```

### Advantages I found

- **Less context switching**: I do not need to hold a full mental model of the project all the time
- **Faster reviews**: The diff is clear and focused
- **Implicit documentation**: The conversations remain as a record of _why_

### What I still do manually

Not everything can be delegated:

- Important architecture decisions
- Final security review
- Complex integration tests

## My current shortcuts

| Action      | Before       | Now                             |
| ----------- | ------------ | ------------------------------- |
| Find file   | `Cmd+P`      | Natural language description    |
| Refactor    | `F2` + manual| Prompt with context             |
| Debug       | Breakpoints  | Describe the symptom            |

## Conclusion

This is not about replacing VS Code completely, but about using the right tool for each task. For surgical, well-defined changes, AI is more efficient. For open-ended exploration and visual debugging, the traditional editor is still valuable.
