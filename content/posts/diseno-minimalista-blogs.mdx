Cuando se trata de blogs técnicos, el diseño debe desaparecer. El contenido es el protagonista.

## La regla del ruido visual

Cada elemento en pantalla que **no es contenido** es ruido. Los bordes excesivos, los gradientes innecesarios, las animaciones que no aportan: todo esto distrae al lector.

### Principios que sigo

- **Tipografía clara**: Una fuente sans-serif para el cuerpo, monospace para el código
- **Contraste suficiente**: Texto claro sobre fondo oscuro, sin ser agresivo
- **Espacio negativo**: Márgenes generosos para que el contenido respire
- **Jerarquía obvia**: Los títulos deben guiar la lectura sin esfuerzo

## Código como ejemplo

Un bloque de código bien presentado vale más que mil palabras:

```css
body {
  background: #0a0a0a;
  color: #e5e5e5;
  font-family: system-ui, sans-serif;
  line-height: 1.7;
}
```

## Lo que evito

1. Barras laterales con widgets innecesarios
2. Pop-ups de suscripción agresivos
3. Más de dos fuentes tipográficas
4. Colores saturados que compiten con el código

> "El buen diseño es tan poco diseño como sea posible." — Dieter Rams

La próxima vez que diseñes un blog técnico, pregúntate: *¿este elemento ayuda al lector o solo me hace sentir bien como diseñador?*
